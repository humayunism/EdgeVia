## What is Edgevia?

Edgevia sits between the internet and your origin server. When traffic exceeds your configured threshold, it activates a **Virtual Waiting Room** вҖ” holding users in a fair, branded queue instead of letting your server crash.

One CNAME change. Automatic SSL. Your business keeps running.

## Features

### рҹӣЎпёҸ The Shield вҖ” Core Proxy Engine
- **Zero-config onboarding** вҖ” add one CNAME, everything else is automatic
- **Auto SSL** вҖ” Let's Encrypt certificates provisioned in under 60 seconds per domain
- **Token bucket rate limiting** вҖ” per-tenant, per-domain, configurable RPS with burst allowance
- **Circuit breaker** вҖ” auto-detects origin failure, stops the cascade before it starts

### рҹҺҹпёҸ Virtual Waiting Room
- Fair **FIFO queue** вҖ” no position jumping, no bots cutting the line
- Real-time position updates via **SSE** (no polling)
- **Fully branded** вҖ” custom HTML/CSS waiting room per site
- **SEO-safe** вҖ” verified crawlers (Google, Bing) bypass the queue automatically

### рҹ“Ҡ Real-Time Dashboard
- Live visitor count, queue depth, RPS gauge вҖ” updated every 500msвҖ“1s
- Inline RPS config вҖ” change rate limits without a page reload
- **Protected revenue counter** вҖ” see what Edgevia saved you in real-time
- Traffic analytics: 7 / 30 / 90 day breakdowns, peak hour heatmaps
- **Event Mode** вҖ” dedicated protection for scheduled flash sales and drops

### рҹҸў Multi-Tenant Architecture
- Full tenant isolation вҖ” separate Redis namespaces, PostgreSQL Row-Level Security
- Unlimited sites per account (Business plan)
- Audit log for every configuration change

### рҹ’і Billing Built In
- Stripe-powered subscriptions with usage metering
- Overage alerts at 80% and 95% of plan limit
- One-click Event Add-on activation
- Self-service plan upgrades and invoice history

### Tech Stack

| Layer | Technology | Why |
|---|---|---|
| **Proxy Engine** | Go 1.22 | goroutines handle 50K+ concurrent connections at sub-ms latency |
| **Management API** | Node.js + NestJS | Modular, typed, decorator-driven вҖ” perfect for business logic |
| **Frontend** | Next.js 14 (App Router) | SSR for first load, real-time via Socket.io |
| **Database** | PostgreSQL + Prisma | ACID compliance for billing and multi-tenant data |
| **Cache + Queue** | Redis 7 | Token buckets, FIFO queues, pub/sub, BullMQ |
| **Internal Comms** | gRPC + Protobuf | Type-safe binary protocol between Go and Node.js |
| **SSL** | Let's Encrypt + autocert | Auto-provision and renew certs per customer domain |
| **Metrics** | Prometheus + Grafana | Operational visibility and alerting |
| **Billing** | Stripe | Subscriptions, usage metering, webhooks |
| **Deployment** | Kubernetes | Auto-scaling, HA, HPA per service |

## Getting Started

### Prerequisites

| Tool | Version |
|---|---|
| Go | 1.22+ |
| Node.js | 20+ |
| pnpm | 9+ |
| Docker + Docker Compose | latest |
| Redis | 7+ |
| PostgreSQL | 15+ |

### Local Development

```bash
# 1. Clone the repository
git clone https://github.com/yourusername/edgevia.git
cd edgevia

# 2. Install Node.js dependencies (all workspaces)
pnpm install

# 3. Start infrastructure (PostgreSQL + Redis)
docker-compose up -d postgres redis

# 4. Set up environment variables
cp apps/api/.env.example apps/api/.env
cp apps/dashboard/.env.example apps/dashboard/.env.local

# 5. Run database migrations
cd apps/api && pnpm prisma migrate dev

# 6. Start all services in parallel (Turborepo)
pnpm dev
```

Services will be available at:
- **Dashboard** вҶ’ http://localhost:3000
- **API** вҶ’ http://localhost:4000
- **Proxy** вҶ’ http://localhost:8080
- **Prometheus** вҶ’ http://localhost:9090

### Go Proxy Only

```bash
cd apps/proxy
go mod download
go run cmd/server/main.go
```

## Security

| Threat | Mitigation |
|---|---|
| Tenant data leakage | PostgreSQL RLS + Redis key namespacing per `tenant_id` |
| JWT token theft | 15-minute access tokens + httpOnly refresh cookies |
| DDoS against Edgevia | Cloudflare in front of Edgevia infrastructure |
| SSL cert exposure | Certs stored encrypted in PostgreSQL, never logged |
| Stripe webhook spoofing | `stripe-signature` header verification on every webhook |
| Bot traffic in queue | User-Agent + behavioral analysis + CAPTCHA on waiting room |
| gRPC interception | mTLS between proxy and API inside Kubernetes |
